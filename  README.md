# ValidatorGen Experiment

Экспериментальный проект для сравнения подходов к реализации кросс-полевой валидации DTO.

В рамках эксперимента сравниваются пять подходов:

1. Рукописный Java-валидатор.
2. Рукописная Jakarta Validation.
3. Сгенерированный прототипом explicit validator.
4. Сгенерированные прототипом Jakarta Validation-артефакты.
5. Валидация через rule engine.

Все подходы проверяются на одном DTO `MoneyTransferDto`, одном наборе правил и одном наборе тестовых экземпляров.

## Основные правила эксперимента

- Все подходы реализуют одинаковый набор правил.
- Все подходы проверяются на одинаковом наборе DTO.
- Результат каждого подхода приводится к единому формату `ViolationView`.
- Сгенерированные артефакты не редактируются вручную после генерации.
- Если сгенерированный код не компилируется без ручных правок, это фиксируется как результат эксперимента.
